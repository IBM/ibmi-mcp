# Quick Start Guide

This guide will get your IBM i MCP Server operational in under 10 minutes. By the end, you'll have a working MCP server with SQL tools connected to your IBM i system.

<Note>
**Prerequisites**: Make sure you have completed the [Installation](/installation) steps before proceeding with this quick start guide.
</Note>

## Step-by-Step Setup

### 1. Verify Prerequisites

<Steps>
<Step title="Check Node.js Version">
  Ensure you have Node.js 20.0.0 or higher installed:
  
  ```bash
  node --version  # Should output v20.0.0 or higher
  ```
</Step>

<Step title="Verify IBM i Access">
  Confirm you can reach your IBM i system and have the following information:
  - IBM i hostname or IP address
  - Valid user credentials
  - Mapepire daemon or gateway running on your configured port (default 8076)
</Step>
</Steps>

### 2. Initial Configuration

<Steps>
<Step title="Clone and Install">
  ```bash
  git clone https://github.com/ajshedivy/ibmi-mcp-server.git
  cd ibmi-mcp-server
  npm install
  ```
</Step>

<Step title="Configure Environment">
  Copy the example environment file and update it with your IBM i details:
  
  ```bash
  cp .env.example .env
  ```
  
  Edit the `.env` file with your specific configuration:
  
  ```bash
  # IBM i Connection
  DB2i_HOST=10.1.1.100           # Your IBM i IP/hostname
  DB2i_USER=MYUSER               # Your IBM i user profile
  DB2i_PASS=mypassword           # Your IBM i password
  DB2i_PORT=8076                 # Mapepire port (usually 8076)
  DB2i_IGNORE_UNAUTHORIZED=true  # For self-signed certificates
  
  # Server Configuration
  MCP_TRANSPORT_TYPE=stdio        # Start with stdio for testing
  MCP_LOG_LEVEL=info              # Logging verbosity
  TOOLS_YAML_PATH=prebuiltconfigs # Location of SQL tool definitions (dir or file)
  ```
</Step>

<Step title="Build the Project">
  ```bash
  npm run build
  ```
  
  <Check>
  You should see successful compilation output with no errors.
  </Check>
</Step>
</Steps>

### 3. First Server Run

<Steps>
<Step title="Start in STDIO Mode">
  Start the server in STDIO transport mode for testing:
  
  ```bash
  npm run start:stdio
  ```
  
  You should see output similar to:
  
  ```
  [INFO] IBM i MCP Server starting...
  [INFO] Transport: stdio
  [INFO] Loading YAML tools from: prebuiltconfigs
  [INFO] Registered YAML tools and resources
  [INFO] Server ready and listening on stdio
  ```
</Step>

<Step title="Verify Connectivity">
  Database connectivity is established on the first tool execution. To verify, run a simple tool (see Inspector section below) such as `system_status` and confirm you receive results. If you see connection errors, double‚Äëcheck your `.env` configuration and ensure your IBM i system is accessible.
</Step>
</Steps>

### 4. Test with MCP Inspector

The MCP Inspector provides a web interface to test your server:

<Steps>
<Step title="Configure Inspector">
  Create the MCP Inspector configuration:
  
  ```bash
  cp template_mcp.json mcp.json
  ```
  
Update (or create) `mcp.json` with your connection details:
  
  ```json
  {
    "mcpServers": {
      "ibmi-server": {
        "command": "node",
        "args": ["dist/index.js"],
        "env": {
          "DB2i_HOST": "10.1.1.100",
          "DB2i_USER": "MYUSER", 
          "DB2i_PASS": "mypassword",
          "DB2i_PORT": "8076",
          "MCP_TRANSPORT_TYPE": "stdio",
          "TOOLS_YAML_PATH": "prebuiltconfigs"
        }
      }
    }
  }
  ```
</Step>

<Step title="Start Inspector">
  ```bash
  npm run inspector
  ```
  
  Open the displayed URL in your browser:
  ```
  üîç MCP Inspector running at http://127.0.0.1:6274
  ```
</Step>

<Step title="Test Tools">
  In the MCP Inspector:
  1. Select your server from the dropdown
  2. Explore available tools in the sidebar
  3. Try running the `system_status` tool
  4. View the results and verify data is returned from your IBM i system
</Step>
</Steps>

## What's Included

Your IBM i MCP Server comes with pre-configured tools organized by category (from `prebuiltconfigs/`):

<CardGroup cols={2}>
  <Card title="System Monitoring" icon="chart-line">
    - **system_status**: CPU, memory, and I/O metrics
    - **system_activity**: Current system activity details
    - **remote_connections**: Established remote connections
    - **memory_pools**: Memory pool sizes and thread utilization
    - **temp_storage_buckets** / **unnamed_temp_storage**: Temp storage usage
    - **http_server**: HTTP server performance metrics
    - **system_values**: Numeric system values
    - **collection_services** / **collection_categories**: Collection Services
    - **active_job_info**: Top CPU consumers in key subsystems
  </Card>
  
  <Card title="Database Operations" icon="database">
    - **execute_sql**: Dynamic SQL execution with safety restrictions (YAML example)
    - **describe_object**: Generate object DDL via QSYS2.GENERATE_SQL
    - Additional discovery/browse/search utilities in `sys-admin.yaml`
  </Card>
</CardGroup>

### Troubleshooting Quick Issues

<AccordionGroup>
<Accordion title="Server won't start">
- Check Node.js version: `node --version`
- Verify build completed: `ls dist/` should show compiled files
- Check environment file: `cat .env` for correct IBM i details
</Accordion>

<Accordion title="Can't connect to IBM i">
- Test network connectivity: `ping your-ibmi-host`
- Verify Mapepire daemon is running on IBM i
- Check firewall rules between your server and IBM i
- Confirm user credentials are valid
</Accordion>

<Accordion title="No tools showing in Inspector">
- Verify YAML files exist: `ls prebuiltconfigs/`
- Check server logs for YAML parsing errors
- Ensure `TOOLS_YAML_PATH` points to correct directory
</Accordion>
</AccordionGroup>
