# Installation Guide

This guide will walk you through installing and setting up the IBM i MCP Server on your system. The server requires Node.js and access to an IBM i system for database connectivity.

## Prerequisites

Before installing the IBM i MCP Server, ensure you have the following requirements met:

### System Requirements

<CardGroup cols={2}>
  <Card title="Node.js Runtime" icon="node">
    **Node.js 20.0.0 or higher** is required for TypeScript and modern JavaScript features
  </Card>
  <Card title="IBM i System" icon="server">
    **IBM i system** with network accessibility and **Mapepire daemon** or gateway configured
  </Card>
</CardGroup>

### Node.js Installation

<Tabs>
<Tab title="macOS">
Install Node.js using Homebrew:

```bash
# Install Homebrew if not already installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js
brew install node

# Verify installation
node --version  # Should be 20.0.0 or higher
npm --version
```
</Tab>

<Tab title="Windows">
Download and install Node.js from the official website:

1. Visit [nodejs.org](https://nodejs.org/)
2. Download the LTS version for Windows
3. Run the installer and follow the setup wizard
4. Verify installation in Command Prompt:

```powershell
node --version  # Should be 20.0.0 or higher
npm --version
```
</Tab>

<Tab title="Linux">
Install Node.js using your distribution's package manager:

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS/RHEL/Fedora
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo dnf install -y nodejs

# Verify installation
node --version  # Should be 20.0.0 or higher
npm --version
```
</Tab>
</Tabs>

### IBM i System Requirements

<Warning>
Ensure your IBM i system is properly configured with network access and the Mapepire daemon or gateway is running on your configured port (default: 8076).
</Warning>

**Required IBM i Components:**
- **Mapepire daemon** or **Mapepire gateway** for database connectivity
- **Network access** from your server to the IBM i system
- **Valid user profile** with appropriate database permissions

## Installation Steps

### 1. Clone the Repository

Download the IBM i MCP Server source code from GitHub:

```bash
git clone https://github.com/ajshedivy/ibmi-mcp-server.git
cd ibmi-mcp-server
```

### 2. Install Dependencies

Install all required Node.js packages:

```bash
npm install
```

<Info>
This will install all dependencies listed in `package.json`, including the MCP SDK, IBM i Mapepire client, and development tools.
</Info>

### 3. Environment Configuration

Create your environment configuration file:

```bash
cp .env.example .env
```

Edit the `.env` file with your IBM i connection details:

```bash
# IBM i DB2 for i Connection Settings
DB2i_HOST=your-ibmi-hostname-or-ip
DB2i_USER=your-ibmi-username
DB2i_PASS=your-ibmi-password
DB2i_PORT=8076
DB2i_IGNORE_UNAUTHORIZED=true

# MCP Server Configuration
MCP_TRANSPORT_TYPE=stdio
MCP_LOG_LEVEL=info
TOOLS_YAML_PATH=prebuiltconfigs
```

<ParamField path="DB2i_HOST" type="string" required>
  The hostname or IP address of your IBM i system
</ParamField>

<ParamField path="DB2i_USER" type="string" required>
  IBM i user profile with database access permissions
</ParamField>

<ParamField path="DB2i_PASS" type="string" required>
  Password for the IBM i user profile
</ParamField>

<ParamField path="DB2i_PORT" type="number" default="8076">
  Port number for the Mapepire daemon or gateway
</ParamField>

<ParamField path="DB2i_IGNORE_UNAUTHORIZED" type="boolean" default="true">
  Whether to ignore SSL certificate validation (useful for self-signed certificates)
</ParamField>

### 4. Build the Project

Compile the TypeScript source code:

```bash
npm run build
```

<Check>
If the build completes successfully, you'll see the compiled JavaScript files in the `dist/` directory.
</Check>

### 5. Verify Installation

Test your installation by running the server in stdio mode:

```bash
npm run start:stdio
```

You should see startup logs indicating the transport and that YAML tools were loaded from your configured path.

<Tip>
Press `Ctrl+C` to stop the server after verification.
</Tip>

### Common Installation Issues

<AccordionGroup>
<Accordion title="Node.js Version Error">
  **Error**: `Error: The engine "node" is incompatible with this module.`
  
  **Solution**: Upgrade to Node.js 20.0.0 or higher:
  ```bash
  node --version  # Check current version
  # Follow the Node.js installation steps above
  ```
</Accordion>

<Accordion title="IBM i Connection Failed">
  **Error**: `Connection failed to IBM i system`
  
  **Solutions**:
  - Verify IBM i hostname/IP is correct and accessible
  - Ensure Mapepire daemon is running on the specified port
  - Check firewall settings between your server and IBM i system
  - Verify user credentials and permissions
  - Confirm the server is actually executing a database tool (connection occurs on first query)
</Accordion>

<Accordion title="Build Failures">
  **Error**: `TypeScript compilation errors`
  
  **Solutions**:
  - Ensure you have the latest dependencies: `npm install`
  - Clear npm cache: `npm cache clean --force`
  - Delete `node_modules` and reinstall: `rm -rf node_modules && npm install`
</Accordion>
</AccordionGroup>
